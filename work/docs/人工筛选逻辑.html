读取表：item_match_result
标志位：human_processed，人工处理过是1，未处理过是0


逻辑：
按照sim_score的排序，相似度高的先处理，展示给人工看，由人标记是否匹配，并标记回item_match_result 表中
两个字段：
首先把human_processed设置为1
如果人工标记为匹配：
human_match = 1 
否则 ：
human_match = 0 

尽量能记录处理人和处理时间，后续追错，排查问题都可能会用到。

注意事项：
系统要支持多人同时处理，有两种方式：
1.每人处理不同的类别，这样不会冲突；
2.多人可以处理不同的类别，这样就需要设计一个任务分配机制，保证大家分到的待匹配列表互相不重叠；
————下面是升级--------
3.允许多人有一定量的重复，互相检查，防止处理错误，同时还能考核处理人的处理质量，防止不称职的人乱标数据污染系统。
    这就需要一个比较复杂的算法来分配任务，检查结果，可以参考一些众包平台的设计论文。
    涉及的问题很多：
    确定哪些商品需要多人对照？
    确定出现冲突时该如何解决？
    如何计算匹配人员的可信程度？

初期建议用1或2。
